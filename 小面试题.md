## 1、String、StringBuffer、StringBuilder的区别
这三者是 Java 中处理字符串的核心类，它们的主要区别在于 **可变性（Mutability）**、**线程安全（Thread Safety）** 和 **性能（Performance）**。

| **特性**   | **String**     | **StringBuffer**         | **StringBuilder**     |
| -------- | -------------- | ------------------------ | --------------------- |
| **可变性**  | **不可变**        | **可变**                   | **可变**                |
| **线程安全** | **安全** (本身不可变) | **安全** (使用 synchronized) | **不安全**               |
| **性能**   | 最慢 (频繁修改时)     | 中等 (有锁开销)                | **最快**                |
| **适用场景** | 少量字符串操作，作为常量   | 多线程环境下的字符串拼接             | 单线程环境下的字符串拼接 (绝大多数情况) |
### String：不可变对象
**原理**：`String` 类被 `final` 修饰，一旦创建，其内容无法更改。当你对 String 做拼接（`+`）或截取操作时，**实际上是创建了一个全新的 String 对象**，旧对象会被垃圾回收。
**优点**：
- **线程安全**：因为不可变，所以多线程并发访问不需要同步。
- **字符串常量池**：利用缓存池机制，节省内存。
**缺点**：
- 如果在循环中频繁进行字符串拼接，会产生大量临时对象，导致内存占用高、性能差（GC 压力大）。
### StringBuffer：线程安全的可变字符序列
**原理**：它是可变的，修改字符串时直接在原对象上操作。它在所有关键方法上都加了 `synchronized` 关键字。
